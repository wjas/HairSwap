# 发型模板图片目录 📸

存放 AI 换发型使用的发型模板图片。

## 📁 当前文件

| 文件名 | 说明 | 用途 |
|--------|------|------|
| `发型 01.png` | 发型模板 1 | 波浪长发 |
| `发型 02.png` | 发型模板 2 | 短发造型 |

## 📋 图片要求

### 基本规格
- **格式**：PNG（推荐）或 JPG
- **尺寸**：建议 512x512 或更高分辨率
- **大小**：单张不超过 10MB

### 内容要求
- ✅ 清晰的发型照片
- ✅ 背景简洁（纯色最佳）
- ✅ 光线均匀
- ✅ 发型层次分明
- ❌ 避免复杂背景
- ❌ 避免人物脸部（只保留发型）

## 🚀 如何添加新发型

### 方法 1：直接复制
```bash
# 从其他位置复制
cp /path/to/hairstyle.png ./发型 03.png
```

### 方法 2：下载保存
1. 准备发型图片
2. 保存到此目录
3. 确保文件名与 `app.js` 配置一致

### 方法 3：使用现有图片
从项目根目录复制：
```bash
cp ../../发型 01.png ./
cp ../../发型 02.png ./
```

## ⚙️ 配置说明

在 `../js/app.js` 中配置发型列表：

```javascript
hairstyleMap: {
  style1: {
    path: 'images/发型 01.png',
    name: '发型 1'
  },
  style2: {
    path: 'images/发型 02.png',
    name: '发型 2'
  }
  // 添加更多发型...
}
```

## 📊 使用火山引擎 API

### 图片顺序
API 调用时图片顺序很重要：
- **第一张**：人物照片（主图，保持脸部）
- **第二张**：发型模板（参考图，提供发型）

### 提示词示例
```
将图 2 的发型换到图 1 上，保持图 1 的脸部、五官、背景等其他元素不变
```

## 💡 最佳实践

1. **统一尺寸**：所有模板使用相同尺寸
2. **高质量**：使用清晰、高分辨率图片
3. **命名规范**：使用中文或英文，避免特殊字符
4. **备份**：定期备份模板图片
5. **测试**：添加新模板后测试效果
